Plan & Design

This project focuses on creating an interactive information system that analyses and visualizes fish population data from publicly available datasets. The system will help researchers, policymakers, and the public understand trends in fish populations, supporting sustainable fishing and conservation efforts. The target audience includes environmentalists, marine biologists, policymakers, and individuals interested in ecological data. The system will be developed using an Agile approach, ensuring flexibility and iterative improvements based on testing and feedback.

The project meets the basic requirements by extracting and cleaning data using Python (Pandas, NumPy) before storing it in a structured format like CSV or JSON. The data will be analysed and represented using Matplotlib, to generate at least two visualizations, such as a bar chart and a line graph, which will highlight trends in fish populations over time. The web-based interface developed using React, HTML, CSS, and JavaScript, will display these visualizations interactively, allowing users to hover over data points, filter results, and adjust ranges for deeper insights.

To meet the advanced requirements, a user survey form will be implemented using JavaScript and Firebase Realtime Database, allowing users to submit feedback, observations, or additional data related to fish populations. The form will collect at least three different types of data (e.g., text responses, numerical values, and dropdown selections), ensuring diversity in user input. Data will be validated before storage, and aggregated survey results will be displayed dynamically. Additionally, a recommendation section will analyse the survey responses and fish population trends to suggest conservation measures or policy adjustments.

The project considers ethical and social aspects, ensuring data integrity, transparency, and accessibility. Since the data originates from publicly available sources, proper attribution will be maintained. The system aligns with the brief‚Äôs objective of using open data to support decision-making, innovation, and societal benefits. By integrating Python for data analytics, JavaScript for interactivity, and Firebase for real-time storage, the project creates an engaging platform that enhances public understanding of fish populations and sustainability.

System Design Flowchart and Architecture

Create

  • Week 1: The project began with selecting a reliable fish population dataset. The dataset was obtained in CSV format and explored for structure, identifying key attributes like region, year, and population count. Basic statistical summaries were generated to understand its scope and limitations.
  • Week 2: Data cleaning commenced using Python‚Äôs Pandas library. Missing values were handled through imputation, and non-numeric characters were removed using the replace() function. Duplicate entries were dropped, and data types were standardized to ensure consistency for analysis. The cleaned dataset was stored in a structured format for further processing.
  • Week 3: Data analysis began with an exploration of fish population trends. Using Matplotlib, initial visualizations were created, including line graphs for population trends and bar charts for regional comparisons. These visualizations helped identify potential insights, such as population declines in certain areas.
  • Week 4: A web-based interface was planned and structured using HTML, CSS, and JavaScript. The front-end framework was set up to ensure a user-friendly design, with a navigation menu and sections for data visualization, interactive charts, and user input. Basic page elements were styled to enhance readability and accessibility.
  • Week 5: The first implementation of the interactive information system was developed. Fish population data was displayed dynamically using JavaScript, and basic event listeners were implemented to enable user interactions such as filtering data by ocean region. This step laid the foundation for more advanced interactive features.
  • Week 6: Advanced interactivity was introduced, including hover effects for data points, tooltips for displaying additional details, and filtering options for users to select different time periods. This was achieved using D3.js and Chart.js, improving user engagement with the data.
  • Week 7: A user survey form was designed to collect feedback and additional data from visitors. The form included at least three types of input (e.g., numerical values, multiple-choice selections, and text responses) and was validated using JavaScript before submission. Firebase Realtime Database was integrated to store survey responses.
  • Week 8: The database connection was refined to ensure seamless data storage and retrieval. Firebase event listeners were used to display newly submitted survey responses dynamically on the webpage. Error handling mechanisms were implemented to improve data reliability.
  • Week 9: A recommendation engine was developed using Python. The system analysed trends in fish population data and user survey inputs to generate meaningful insights. For example, if survey results indicated concerns about overfishing, the system highlighted declining populations and suggested conservation efforts.
  • Week 10: Testing and debugging were conducted to ensure system reliability. Python scripts were tested for data processing accuracy, while JavaScript functions were debugged for proper interactivity. The website was tested on multiple browsers and devices to ensure compatibility.
  • Week 11: Performance optimization was carried out to improve load times and responsiveness. Large datasets were processed more efficiently by aggregating values before visualization. Pagination and lazy loading were introduced to prevent browser slowdowns when displaying survey data.
  • Week 12: Final refinements were made based on user feedback. The UI was polished, additional tooltips were added for usability, and final bug fixes were implemented. The project was thoroughly reviewed to ensure all requirements from the brief were met before submission.

    • Throughout the development process, extensive testing was conducted to validate each component. Unit testing was performed on Python scripts to confirm that data extraction, cleaning, and transformation processes worked as expected. Various test cases were considered, such as inputting datasets with missing values, incorrect formats, and duplicate entries to evaluate how well the system handled them. Visualizations were tested by applying different data subsets and ensuring that charts updated dynamically. Additionally, cross-browser testing was conducted to ensure that the interactive elements functioned correctly in Chrome, Firefox, and Edge. Usability testing was also performed, where sample users provided feedback on the website‚Äôs intuitiveness and ease of navigation.

    One of the most challenging aspects of the project was handling real-time survey data updates. Initially, Firebase successfully stored responses, but retrieving and displaying the latest entries dynamically caused delays. This issue was resolved by using Firebase event listeners to trigger automatic updates whenever new data was added. This ensured that users could immediately see how their inputs influenced the dataset. Another major challenge involved optimizing large datasets for visualization. Initially, rendering thousands of data points caused significant slowdowns. To solve this, data aggregation techniques were applied, grouping population values over time instead of displaying raw entries. Additionally, pagination and lazy loading were introduced for smoother performance. The algorithm developed for the analytics section aimed to summarize fish population trends while incorporating user feedback from the survey. The algorithm first aggregated population data by region and year, calculated percentage changes, and identified areas with significant declines. It then cross-referenced survey responses to generate personalized insights. For example, if a user expressed concerns about declining fish populations in the Pacific Ocean, the system would highlight relevant data and suggest conservation efforts.
